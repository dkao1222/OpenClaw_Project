# Google Play Data Safety

## 繁體中文

- Data collected: Advertising ID / Device ID、App interactions、Diagnostics、Crash logs、Performance data，實際項目依整合 SDK 更新。
- Data shared: 若廣告 SDK 接收資料用於 advertising or marketing、analytics、fraud prevention 或 security，需標示 sharing。
- Security practices: 所有傳輸必須使用 HTTPS 或等效現代加密。
- User control: 必須提供 privacy policy URL；若收集個人與敏感資料，需提供清楚揭露與必要 consent。
- Children / Families: 若目標受眾包含兒童，廣告 SDK、資料收集與 personalized ads 必須符合 Google Play Families policies。

## English

- Data collected: Advertising ID / Device ID, App interactions, Diagnostics, Crash logs, and Performance data, updated according to actual SDK integration.
- Data shared: If an ad SDK receives data for advertising or marketing, analytics, fraud prevention, or security, mark sharing.
- Security practices: all transmission must use HTTPS or equivalent modern encryption.
- User control: provide a privacy policy URL; for personal and sensitive user data, provide clear disclosure and required consent.
- Children / Families: if the target audience includes children, ad SDKs, data collection, and personalized ads must comply with Google Play Families policies.
