# Implementation Readiness Checklist

## 繁體中文

- Ready: platform direction、MVP scope、core loop、screen list、HUD、failure/retry flow、visual/audio feedback 都已定義。
- Ready: development backlog 可拆成 project shell、gameplay、UI、feedback、QA 與 final package。
- 注意: development 仍需使用 handoff_to_game_development.md 作為唯一入口，並等待使用者確認。
- Block if missing: icon contract、operation feel、canvas artifact、game systems contract、GDD acceptance criteria 任一缺失都不可進入 development。

## English

- Ready: platform direction, MVP scope, core loop, screen list, HUD, failure/retry flow, and visual/audio feedback are defined.
- Ready: the development backlog can split into project shell, gameplay, UI, feedback, QA, and final package.
- Note: development must use handoff_to_game_development.md as the only entry point and wait for user confirmation.
- Block if missing: icon contract, operation feel, canvas artifact, game systems contract, or GDD acceptance criteria must prevent development.
